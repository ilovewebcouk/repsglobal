## Why the current output looks weak

The AI hero is currently generated by `generateHeroFromAi` in `src/lib/shop-front/hero.functions.ts` using:

- **Model:** `google/gemini-3.1-flash-image` (the *fast* Nano Banana 2 tier — optimised for speed, not fidelity).
- **Reference:** only the trainer's avatar (often a low-res, badly-lit selfie).
- **Prompt:** a short rules list (REPS wordmark, 9:16, "editorial") with no cinematography direction, no lighting, no lens, no mood, no colour grade.
- **No style anchor:** the model has no visual reference for what "REPs cinematic hero" actually looks like, so it averages toward generic stock-gym output.

The James Carter reference (`/assets/coach-james-coaching-*.jpg`) was hand-crafted with the premium imagegen pipeline + a rich cinematic prompt + REPS wordmark rules. We never gave the trainer-facing generator the same treatment.

## Fix

### 1. Upgrade the model

Switch from `google/gemini-3.1-flash-image` → **`google/gemini-3-pro-image`** (the pro tier — same family, materially higher fidelity on skin, fabric texture, embroidery detail, and lighting). Slower and a bit more expensive per call, but this is a once-per-trainer hero, not a high-volume call.

### 2. Send a style anchor, not just the avatar

Pass **two** reference images to the model:

1. The trainer's `avatar_url` — for face/build/skin tone match (existing).
2. The locked James Carter cinematic hero (`/assets/coach-james-coaching-*.jpg`) — as a **style reference** so the model anchors lighting, grade, framing, and wordmark execution to our brand standard instead of guessing.

The prompt tells the model explicitly: "Match the face from image 1. Match the lighting, colour grade, framing and REPS wordmark execution from image 2."

### 3. Rewrite the system prompt with cinematography direction

Replace the current short rules block with a full editorial brief covering:

- **Lighting:** soft directional key, warm rim light, cinematic falloff — never flat phone-flash.
- **Lens & framing:** 35mm or 50mm equivalent, shallow depth of field, subject filling 60–70% of frame, headroom for hero copy on the left.
- **Colour grade:** warm highlights / cool shadows, slight teal-orange editorial wash, filmic contrast.
- **Wardrobe:** premium athletic fit, neutral/earth tone, REPS wordmark rules (ALL CAPS, white, real stitched embroidery, medium weight, left chest or centre chest — never a flat decal).
- **Environment:** premium private studio / industrial gym / golden-hour outdoor, never bright commercial chain-gym lighting.
- **Pose:** intentional, in-action or confident still — never a posed gym-bro selfie.
- **Negative list:** no text overlays, no graphic logos other than REPS, no watermarks, no studio backdrop, no smiling-to-camera stock pose, no flipped/mirrored shots.

### 4. Add a "Style" preset in the editor (small UI add)

In `HeroImageEditor.tsx` AI tab, add three style chips above the prompt field:

- **Editorial** (default — golden hour, warm, James Carter style)
- **Studio** (clean, dark, contrasty, magazine cover)
- **In-action** (mid-rep, motion, sweat detail)

The chip prepends a style block to the prompt sent to the model, so trainers don't need to know cinematography vocab to get a good result.

### 5. Surface a "Regenerate" button on the result

If the first pass isn't quite right, one click re-rolls with the same brief + style — keeping the user in the AI tab instead of forcing them back through the upload flow.
