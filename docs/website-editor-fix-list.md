# Website Editor ‚Äî Prioritised Fix List

_Companion to `docs/website-editor-audit-final.md`. Grouped by severity, then by area. Each row is a discrete change ready to hand off to the next build turn._

## Legend

- **P0** ‚Äî security, data loss, publish/discard corruption, cross-tenant leak. Must ship before next public launch.
- **P1** ‚Äî wrong state shown to trainer, broken pipelines, a11y block, silent staleness.
- **P2** ‚Äî copy, empty-state polish, perf, dead code.

## Suggested ship order

1. **Migration bundle (P0-02, P0-04, P0-05, P1-15, P1-16, P2-16)** ‚Äî one PR, one destructive migration, tight blast radius.
2. **Cross-tenant hijack (P0-01)** ‚Äî pure server-fn change, no schema.
3. **Dirty-tracking coverage (P0-03)** ‚Äî trigger migration + `getMySectionDiff` extension + UI (sidebar + dashboard NeedsAttention row).
4. **Remove the live-fallback (P1-01)** ‚Äî after P0-02/03 land, so pros aren't served stale drafts.
5. Then the P1 stack in the order listed.
6. P2s in batches by area (image pipeline, then dialogs, then copy).
