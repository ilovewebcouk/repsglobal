# Phase 2.0 — Tier scope (LOCKED 2026-06-14, Pass A)

## Verified — £99/yr
Trust layer only:
- Discoverable profile + verified badge
- Enquiry form on `/pro/$slug` → **email forward** to the pro
  - From: `notify.dogboss.io`, Reply-To = lead's email (pro replies straight from their inbox back to lead)
  - REPs is just the relay
- **No** dashboard inbox, **no** leads pipeline, **no** "Enquiries"/"Leads" sidebar entry
- Visiting `/dashboard/leads` shows the `_pro` UpgradePanel (no redirect)

Verified sidebar (only): Dashboard, Public Profile, Shop-front, Verification, Education & CPD, Settings.

## Pro — £59/mo Founding
Full operating system:
- Enquiries from `/pro/$slug` land **straight in the Leads pipeline** at `/dashboard/leads` (stage = `new`). No intermediate triage inbox.
- Pro also gets the notification email, with CTA → `/dashboard/leads`
- Source of truth = the pipeline UI

## Studio — £149/mo
Waitlist only — mirrors Pro nav for now.

## Code map
- Submit fn: `src/lib/enquiries/enquiries.functions.ts::submitEnquiry`
  - branches via `has_active_tier(_user_id, ['pro','studio'])` to pick CTA url
  - always passes `replyTo: sender_email` so Verified can reply straight to the lead
- Email send pipeline supports `replyTo` → `payload.reply_to` in pgmq:
  - `src/lib/email/send.server.ts`
- Leads route: `src/routes/_authenticated/_professional/_pro/dashboard_.leads.tsx` (under `_pro` gate)
- `/dashboard/enquiries` route + `src/components/enquiries/*` were DELETED in this pass.

## Rules
- Never reintroduce a separate "Enquiries" inbox for Verified or Pro.
- A single H1 per dashboard page — page title comes from `DashboardShell title=`; inner section labels are `h2`.
- Out of scope for Pass A (locked): calendar, bookings, messages, reviews.
